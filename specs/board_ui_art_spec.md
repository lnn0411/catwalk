# 棋盘 UI 按钮/弹窗 · 美术需求与切图规格 v1.0

> 需求方：棋盘（猫咪合合乐）玩法组 ｜ 2026-07-22
> 风格权威：Style Bible v2.2 §3.1（配色以 `autoload/Palette.gd` 为准）
> 质感基准件：`assets/art/ui/incubation/components/btn_confirm_name.png`（孵化页确认按钮，即门票弹窗"知道了"）——新按钮与它同系

---

## 1. 背景与问题

棋盘场景现用的通用按钮资源 `btn_primary.png` / `btn_secondary.png` 仅 **200×40px**，
实际被拉伸到 220×60 ~ 240×64 逻辑尺寸使用，且未做九宫格切图 →
整图拉伸导致**模糊 + 圆角变形**。弹窗底 `popup_bg.png`（400×300）拉到 560×500 同理。
本需求替换这三件资源并补齐按钮状态态，代码侧点位已统一收口
（`S14_BoardGame._apply_btn_art / _apply_dialog_panel_art`），资源到位后仅换路径+配九宫格参数。

## 2. 风格基调

- 治愈系猫咪手游，奶油纸质感；**圆角胶囊按钮**，描边 + 微投影，与 `btn_confirm_name` 同族
- 配色严格取自 Style Bible：
  - 主按钮（primary）：填充 `AMBER #F2C572`，描边 `AMBER_PRESS #C4894A`，文字为代码叠加（`TEXT_PRIMARY #4F453C`），**按钮图内不要带任何文字**
  - 次按钮（secondary）：填充 `PAPER_CREAM #F6EFE2`，描边 `BORDER #EFE4D6` 或 `AMBER_PRESS` 细线
  - 弹窗底：`PAPER_CREAM` 纸质卡片，圆角 ≥40px（@2x），可带轻微纸纹/猫爪暗纹（不超过 5% 对比度，不干扰文字）
- 禁止：高饱和渐变、重阴影、玻璃拟态——与全局风格冲突

## 3. 资产清单（共 7 件）

### 3.1 按钮（6 件）

| 文件名 | 说明 | 画布尺寸 @2x | 备注 |
|---|---|---|---|
| `btn_primary_n.png` | 主按钮·常态 | **480×128** | 琥珀填充+深琥珀描边 |
| `btn_primary_p.png` | 主按钮·按下 | 480×128 | 填充加深至 `#C4894A` 系，整体下沉感（高光收窄或底边压缩 4px） |
| `btn_primary_d.png` | 主按钮·禁用 | 480×128 | 去饱和 + 60% 明度，或灰化 `#D8CFC2` |
| `btn_secondary_n.png` | 次按钮·常态 | 480×128 | 奶油填充+细描边 |
| `btn_secondary_p.png` | 次按钮·按下 | 480×128 | 填充加深至 `BG_CEMENT #F2EDE4`+描边加深 |
| `btn_secondary_d.png` | 次按钮·禁用 | 480×128 | 同主按钮禁用处理 |

- hover 态不需要（手游无鼠标；桌面调试复用常态）
- **必须按九宫格设计**：四角圆角区固定，中段横向可无损拉伸
  - 建议安全边距：左右各 **64px**、上下各 **40px**（@2x）为不可拉伸区
  - 交付时在附注里标明你们实际使用的 margin 值，工程按此配 `StyleBoxTexture.texture_margin_*`
- 拉伸范围参考（逻辑像素，@2x 即 ×2）：宽 80~460 / 高 28~72——中段纹理需经得起 2 倍以上横向拉伸（纯色/微噪点安全，规律图案会花）

### 3.2 弹窗底（1 件）

| 文件名 | 说明 | 画布尺寸 @2x | 备注 |
|---|---|---|---|
| `popup_bg_9p.png` | 通用弹窗卡片底 | **800×600** | 九宫格：四边各留 **80px** 不可拉伸区；替代现 `popup_bg.png` 的拉伸用法 |

- 使用场景：门票不足面板（560×500 逻辑）、狂欢二选一（460×高自适应）、周末委托、捣蛋日奖励二选一、升档说明——同一张图覆盖全部
- 顶部可预留装饰位（猫耳/爪印压角），但四角 80px 安全区内完成，避免拉伸变形

## 4. 投放路径与接入

```
assets/art/ui/buttons/   ← 6 张按钮（沿用现目录，新文件名，旧 btn_primary/secondary.png 保留待代码切换后删除）
assets/art/ui/panels/    ← popup_bg_9p.png
```

- 格式：PNG-24 带透明通道；`.import` 文件由工程生成提交，美术不用管
- 接入点（工程侧一次改完，美术无需改代码）：
  - `S14_BoardGame.gd` 顶部 `TX_BTN_PRIMARY/TX_BTN_SECONDARY` 常量 → 三态字典
  - `_btn_style()` 补 `texture_margin_*`（按交付 margin 值）
  - `_apply_dialog_panel_art()` 换 `popup_bg_9p.png` + margins

## 5. 验收标准

1. 真机 1080×2400 下：按钮四角圆弧无变形、无模糊（对照 `btn_confirm_name` 同屏不违和）
2. 同一张按钮图在 80×28（狂欢小按钮）和 460×64（弹窗宽按钮）两种极端尺寸下中段无花纹断裂
3. 三态在真机点按可明显区分（常态/按下反馈/禁用一眼可辨）
4. 弹窗底在 460×300 与 560×500 两种规格下四角一致
5. 色值抽样与 Style Bible §3.1 偏差 ΔE < 3

## 6. 优先级与数量

- P0（本次必交）：主按钮三态、弹窗底 → 覆盖门票面板/结算/救局的主路径
