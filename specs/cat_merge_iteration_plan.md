# 「猫咪合合乐」迭代优化方案 v1.0

> 基于 2026-07 游戏设计与数值审计产出
> 涉及模块：`scripts/board_game/*`、`scenes/S14_BoardGame.gd`、`autoload/TicketManager.gd`、`tools/econ_sim/*`
> 文档状态：待评审

---

## 0. 方案总览

| 批次 | 主题 | 内容 | 建议节奏 |
|---|---|---|---|
| **M1 热修** | 规则/数值漏洞封堵 | 撤销扣费、奖励刷取、死局误判、掉落倒带、关卡奖励兑现 | 1 个迭代（1 周） |
| **M2 体验** | 奖励层级 + 狂欢重做 + 预警 | 奖励梯度表、狂欢二选一、捣乱预警前置 | 1–2 个迭代 |
| **M3 长线** | 目标感与内容变化 | 胜场里程碑、每日变异规则、订单局 | 2–3 个迭代 |
| **M4 校准** | 数值实测与回归 | 通关率蒙特卡洛、econ_sim 对齐、埋点 | 与 M2/M3 并行 |

**设计原则（全方案适用）**
1. 不推翻现有框架：短局制（20 次生成器）、三链、捣乱/狂欢的骨架保留，只修正回报与兑现。
2. 奖励只升不降：任何改动不得让玩家现有收益变差（安慰奖不动，向上拉开梯度）。
3. 先兑现承诺，再加新内容：文案已宣传的（关卡奖励更丰厚）优先于新系统。
4. 所有数值改动必须过 econ_sim 回归（A7/A8 断言 + 新增 A11 通关率断言）。

## 1. M1 热修批次（P0，必须全部完成后才可做后续批次）

### 1.1 撤销扣钻石接入

**现状**：`BoardGame.undo()` 在免费 3 次用尽后不做拦截（`BoardGame.gd:305`），UI 层扣费为 TODO（`S14_BoardGame.gd:672`），实际无限免费撤销。星级评价、难度、钻石消耗点全部失真。

**方案**
- `S14_BoardGame._on_undo_pressed()`：
  - `get_undo_cost().diamond_cost == 0` → 直接 `board.undo()`。
  - `> 0` → 弹确认框「消耗 10💎 撤销上一步？」，`CurrencyManager.spend_diamond(10)` 成功后才调用 `board.undo()`；余额不足弹「钻石不足」并引导商店。
- 按钮文案：免费期 `↩ 撤销 (n)`，付费期 `↩ 撤销 10💎`；不可撤销时置灰。
- `BoardGame.undo()` 增加防御：`undo_free_count <= 0` 时要求调用方传入 `paid: bool = false` 确认参数（纯逻辑层兜底，防止未来 UI 绕过）。

**数值**：免费 3 次/局、10 钻/次维持不变（日免费钻石 ~15，付费撤销 1–2 次/天是合理压力）。

**验收**
- 第 4 次撤销必扣 10 钻；余额不足时撤销失败且棋盘无变化。
- 自检脚本 `tests/board_game_selfcheck.gd` 新增用例：免费额度耗尽后无支付调用 `undo()` 返回 false。

### 1.2 封堵「撤销-重合成」奖励刷取

**现状**：副链 ⭐3 在**合成时**发 `sub_chain_completed`，UI 即时入库小鱼干（`S14_BoardGame.gd:759`）；撤销后重合可重复领取。兴奋值同样不随撤销回退，可反复拉满狂欢。

**方案**
- **奖励结算后移**：小鱼干奖励从 `_on_sub_chain_completed`（合成信号）移到 `sub_chain_exit()` 成功时结算（与代码注释的原始意图一致，`BoardGame.gd:160`）。`sub_chain_completed` 信号仅用于 UI 提示「可走出口」。
- 出口本身不可撤销（物品已移出棋盘 + 返还生成器为一次性），天然防刷。
- **兴奋值回退**：undo_stack 的 merge 快照增加 `excitement_before` 字段；`_apply_undo("merge")` 时恢复。连击计数一并重置（`_combo_count`、`_last_merge_time` 清零，宁可少给不可多给）。
- 每局小鱼干类奖励设硬上限兜底：`SUB_EXIT_REWARD_MAX_PER_GAME := 2`（对应最多两次出口）。

**验收**：撤销→重合成 100 次，副链奖励入库次数 ≤ 出口实际使用次数；兴奋值在撤销后与合并前一致。

### 1.3 死局判定纳入副链出口自救

**现状**：`_check_deadlock()` 只看可合并对与生成器余量（`BoardGame.gd:706`），棋盘上存在副链 ⭐3 且出口未用时仍会判负——玩家本可用出口换 +2 次生成器盘活。

**方案**：判负条件追加一条豁免：
```
if not sub_chain_exit_used and _board_has_sub_star3():
    return  # 还有出口自救手段，不判负
```
同时 UI 在该状态下高亮副链 ⭐3 与出口按钮（脉冲动画），提示玩家这是最后手段。

**验收**：构造「无可合并 + 生成器 0 + 存在副链⭐3 未用出口」局面，游戏不判负；走出口后生成器 +2 恢复可玩；出口已用过时正常判负。

### 1.4 掉落序列索引改用点击计数

**现状**：掉落索引 = `GENERATOR_TOTAL - generator_remaining`（`BoardGame.gd:275`），出口返还 +2 使索引回退 2 格，序列重放，可能重复命中副链位（第 5/10/15/20），破坏「每局固定 4 个副链掉落」的确定性承诺，且主链实际供给低于账面。

**方案**
- 索引改用 `generator_click_count`（只增不减）：`index = clampi(generator_click_count, 0, DROP_SEQUENCE.size() - 1)`。
- 超出 20 次的点击（出口返还部分）：追加规则「第 21 次起全部产出主链」，明确写入 DROP_SEQUENCE 之外的兜底逻辑（现有 clamp 到 index 19 会出副链，改为返还的点击固定出主链——返还本来就是给主链冲刺用的）。
- 撤销生成器点击时 `generator_click_count` 保持不减（现状即如此，维持防捣乱规避特性），序列因此也不会因撤销漂移。

**数值影响**：主链供给下界从「可能 14–16」修正回稳定 16+返还 2 = 18；LV3 最坏情况（捣乱 −6）净余量从 −2 修正为 0~+2，配合三连合不再出现数值性无解局。**此项是 1.6 难度校准的前置。**

**验收**：任意时点使用出口返还，后续掉落中副链总数恒等于 4；返还的 2 次点击恒出主链。

### 1.5 关卡奖励兑现

**现状**：升档弹窗与 `get_level_config().reward_desc` 承诺 LV2/LV3 奖励更丰厚（`BoardGameData.gd:126`），但 `roll_reward()` 无关卡参数，三档共用一张平权表；LV3 三星额外猫罐头未实现。

**方案**：`BoardRewardSystem.roll_reward(level: int)` 按关卡分表：

| 奖励 | 类型 | LV1 权重 | LV2 权重 | LV3 权重 |
|---|---|---|---|---|
| 小鱼干 ×1 | snack | 30 | 24 | 18 |
| 猫罐头 ×1 | snack | 20 | 22 | 24 |
| 逗猫毛线团 ×1 | toy | 20 | 20 | 18 |
| 逗猫棒 ×1 | toy | 15 | 15 | 14 |
| 猫爬架 ×1 | decoration | 8 | 10 | 12 |
| 樱花树 ×1 | decoration | 5 | 6 | 9 |
| 猫罐头大礼包 ×3 | snack_pack | 2 | 3 | 5 |

- 设计意图：LV3 装饰类合计权重 21%（LV1 为 13%）、大礼包 5%，兑现「装饰+礼包」承诺；小鱼干占比随关卡下降，缓解奖励疲劳。
- **LV3 三星追加**：`rating >= 3 and level == LV3` → 猫罐头×1（每局可得，非一次性），兑现 reward_desc 文案。
- B6 转化上限不变（cat_tree≤3、cherry≤1，超出 → 100 金币）；econ_sim 回归确认 90 天金币注入增幅 ≤ 15%（预估 LV3 玩家装饰产出 13%→21%，超上限部分转金币，A8 断言阈值同步放宽 15%）。

**验收**：三档各 roll 10000 次，分布与表格误差 < 1.5%；LV3 三星通关必得追加猫罐头。

### 1.6 捣乱目标权重反转（难度去尖刺）

**现状**：捣乱偷 ⭐1/⭐2，且 ⭐2 权重双倍（`BoardGame.gd:403`）——伤害最大的目标反而最容易被偷，LV3 最坏损耗 −6 单位贴着无解线。

**方案**：权重反转为 ⭐1:2 / ⭐2:1。最坏期望损耗从 −6 降至约 −4，配合 1.4 后 LV3 最坏净余量稳定为正。捣乱的情绪冲击保留（还是会偷），但不再制造数值死局。看广告找回的价值仍在（省 3–4 个单位 + 省点击数冲三星）。

**验收**：蒙特卡洛（见 M4）LV3 无广告无撤销贪心策略通关率 ≥ 75%。

## 2. M2 体验批次（P1）

### 2.1 奖励层级重建

**现状**：放弃安慰奖 = 死局安慰奖 = 副链奖励 = **首次三星里程碑** = 小鱼干×1。成就感层级完全扁平。

**方案**（自下而上拉开梯度，安慰奖不动）：

| 场景 | 现奖励 | 新奖励 | 定位 |
|---|---|---|---|
| 主动放弃 | 小鱼干×1 | 小鱼干×1（不变） | 保底安慰 |
| 死局失败 | 小鱼干×1 | 小鱼干×1（不变） | 保底安慰 |
| 副链 ⭐3 出口（每局首次） | 小鱼干×1 | **猫罐头×1** | 路线选择奖励 |
| 副链出口（每局第二次） | 无 | 小鱼干×1 | 清格补偿 |
| 首次三星通关（一次性） | 小鱼干×1 | **猫罐头大礼包×3 + 💎20** | 里程碑高光 |
| LV3 三星通关（每局） | 无 | 猫罐头×1（见 1.5） | 高难高报 |

- 结算弹窗按稀有度分级演出：snack 普通弹窗；decoration/礼包走 `Juice.pattern_legendary()` + 稀有度粒子（复用 `items/RarityParticles.gd`）。
- 文案同步：首次三星弹窗改为「🏆 完美通关！猫咪们为你欢呼」。

### 2.2 狂欢系统重做（K8）

**现状**：攒满 100 兴奋值仅可「抵消下一次捣乱」。LV1 只有 1 次捣乱且固定在第 10 击——之后攒满的兴奋值零价值；捣乱时点隐藏，「何时放」是盲赌；新手期恰好是系统表现最差的阶段。

**方案：狂欢改为二选一主动技**。兴奋值满 100 后点击狂欢按钮，弹出双选项卡（3 秒不选默认收起，可稍后再点）：

| 选项 | 效果 | 定位 |
|---|---|---|
| 🛡 猫猫护卫 | 抵消下一次捣乱（现有 K7 逻辑） | 防御，捣乱前使用 |
| ✨ 猫猫帮忙 | 立即免费生成 2 个（不消耗 generator_remaining，产出走当前序列的主链） | 进攻，冲刺/救急 |

- 每局仍限 2 次（`FRENZY_MAX_TRIGGERS` 不变）。
- 「猫猫帮忙」产出恒为主链 ⭐1×2（不走副链位，避免与 1.4 序列规则纠缠），净供给 +2 单位/次——与三连合同量级，不破坏难度框架（econ_sim 回归确认通关率增幅 ≤ 5pp）。
- 已选「护卫」但本局不再有捣乱点时：局末未消耗的护卫自动折算小鱼干×1（不让玩家的选择变废）。
- 兴奋值满后继续合并：溢出部分的 50% 计入下一管（`excitement_overflow_carry := 0.5`），缓解溢出浪费、鼓励连续合并。

### 2.3 捣乱预警前置

**现状**：`mischief_warning` 仅在触发瞬间闪红 0.4s，玩家无反应窗口，狂欢防御选项形同盲赌。

**方案**
- 捣乱触发点前 **2 次点击**开始预警：棋盘边缘出现猫爪剪影 + 目标区域微震（不指明具体格子，保留紧张感）；生成器按钮角标「😼 猫猫蠢蠢欲动…」。
- 预警期内玩家的合理对策：把 ⭐1/⭐2 尽快合掉、或放狂欢护卫。捣乱从「随机惩罚」变成「可应对的压力事件」——这是本批次趣味性提升的核心。
- 实现：`_advance_mischief_trigger()` 已维护 `mischief_pending_trigger`，新增信号 `mischief_forewarning(clicks_left: int)` 在 `generator_click_count == pending - 2` 与 `pending - 1` 时发出。

### 2.4 连击显示修正 + 连击奖励微调

- 修 off-by-one：`combo_triggered.emit(_combo_count)` 移到自增之后（`BoardGame.gd:181`），首个连击显示「连击 ×2」。
- 连击窗口 3.0s 维持；连击兴奋值从固定 +4 改为 `+4 × min(combo, 3)`（封顶 +12），给连续快合一个可感知的爽点，配合 2.2 的溢出结转不浪费。

## 3. M3 长线批次（P1/P2）

### 3.1 胜场里程碑（15 胜后的目标感）

**现状**：LV3（15 胜）后无任何棋盘侧进度，中活跃玩家 3–4 天到顶。

**方案**：累计胜场里程碑表（持久化于 `LevelStateManager` meta，UI 在棋盘入口显示进度条）：

| 累计胜场 | 奖励 | 备注 |
|---|---|---|
| 25 | 💎30 | |
| 50 | 猫罐头大礼包 + 💎30 | |
| 100 | 限定装饰「毛线王座」+ 称号「合合小能手」 | 复用床具链 ⭐5 美术 |
| 200 | 💎80 + 限定明信片×1 | 接明信片收集系统 |
| 300+ | 每 100 胜 💎50（循环） | 长尾 |

- 钻石总注入：前 100 胜共 💎60，摊到 ~25 天，日均 +2.4，占日免费钻石 15 的 16%，在安全线内；econ_sim 增加对应收入源回归。

### 3.2 每日变异规则（Daily Twist）

每天从池中随机 1 条规则挂在当日所有对局上（入口明示，规则透明）：

| 变异 | 效果 | 补偿/代价 |
|---|---|---|
| 毛线日 | 毛线格 +2 | 解格兴奋值 20→30 |
| 捣蛋日 | 捣乱 +1 次 | 通关奖励 roll 两次取其一（玩家选） |
| 慷慨日 | 初始主链 +2 | 三星线收紧：剩余 ≥5 |
| 连击日 | 连击窗口 3s→5s | 无 |
| 双子日 | 主副链互换出口奖励 | 无 |

- 实现为 `get_level_config()` 上的 modifier 叠加层，不动基础配置；变异池配置化（JSON），便于运营调池。
- 目的：用最低内容成本打破「局局相同」，让老玩家每天有一个新决策维度。

### 3.3 订单局（周期玩法，M3 后半）

每周末开放「猫咪委托」模式（消耗门票不变）：通关条件从「主链 ⭐5」替换为组合订单，例如：
- 「交付 零食链⭐3 ×2 + 穿戴链⭐4 ×1」
- 「在 16 次生成内交付 床具链⭐4」

订单从模板池按周轮换；奖励 = 普通局 roll + 委托猫好感 +5（接 `interaction` 系统的 `board_snack_affection` 通道）。三条链从「换皮」变成「订单素材」，首次产生链间差异价值。**此项工作量最大（新胜利判定 + 订单 UI），可独立排期，砍掉不影响 M1/M2。**

## 4. M4 校准批次（与 M2/M3 并行）

### 4.1 通关率实测（替换拍脑袋的 clear_rate=0.9）

- 在 `tests/` 下新增 `board_montecarlo.gd`：贪心策略 bot（优先合并最高星对、空间不足时优先清副链）× 每关卡 5000 局，输出分关卡通关率 / 平均点击数 / 三星率 / 死局原因分布。
- 目标值：LV1 ≥ 92%、LV2 ≥ 85%、LV3 ≥ 75%（无广告无付费撤销）；三星率 LV1 15–25%。
- 实测值回填 `tools/econ_sim/config/params.json` 的 `board_game.clear_rate`（改为按关卡分档），A8 金币注入断言随之重校。
- 新增断言 **A11**：分关卡通关率落在目标区间；每次改棋盘数值必跑。

### 4.2 埋点（上线后验证设计假设）

| 埋点 | 验证的假设 |
|---|---|
| 每局：关卡/胜负/点击数/星级/撤销次数(免费/付费)/狂欢选项/出口使用 | 难度曲线、狂欢二选一的选择率 |
| 捣乱预警后的玩家行为（合并加速？放护卫？） | 2.3 预警是否产生决策 |
| 门票获取分来源 + 当日消耗率 | A7 断言与真实行为对齐 |
| 15 胜后 7 日留存 vs 15 胜前 | 3.1 里程碑有效性 |

### 4.3 econ_sim 对齐

- A7 门票断言当前多画像 FAIL（GDD 预期表与实现漂移）：以 `TicketManager.gd` 常量为准重写断言阈值，GDD §速查表同步修订。
- M2/M3 新增收入源（里程碑钻石、LV3 三星罐头、护卫折算小鱼干）全部进 sim。

## 5. 风险与回滚

| 改动 | 风险 | 缓解 |
|---|---|---|
| 1.1 撤销收费生效 | 习惯免费撤销的玩家负反馈 | 生效首周弹一次性补偿（💎30 + 说明「撤销规则修复」）|
| 1.6+2.2 难度下调叠加 | 通关率过高 → 奖励通胀 | A11 断言卡上限（LV1 ≤ 96%）；超线则回调「猫猫帮忙」为 +1 |
| 2.2 狂欢改动 | 存档兼容（frenzy_cancel_pending 语义变化） | 序列化新增 `frenzy_mode` 字段，旧档缺省视为护卫 |
| 3.1 钻石注入 | 通胀 | 已按日均 +2.4 控制，sim 回归 |
| 奖励分表 | 老玩家库存结构突变 | 只调权重不删条目，B6 上限兜底 |
