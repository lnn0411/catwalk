## 0. 审计结论摘要（为什么要改设计而不是只调数值）

econ_sim 四个红断言（A2/A3/A4/A6）不是参数没调好，而是四个**结构性设计缺陷**的
症状。逐条对应 GDD v3.0 §1.3 五大设计支柱：

| # | 缺陷 | 违背的支柱 | 症状断言 |
|---|------|-----------|---------|
| 1 | 孵化循环零决策（串行自动灌注、蛋无差异、四槽=队列皮肤） | 制作人判例「核心操作每步零决策」（v3.3 棋盘 K10 同款诊断） | — |
| 2 | 节奏两端皆坏：递增费率使低活跃饿死（800/日，5.3天/蛋）、高活跃撑死（9200/日，2蛋+/日） | 支柱1「每一步都在养猫」——前1000步系数0.3恰恰最不养猫 | A4、A6 |
| 3 | 孵化洪水 → 包满刚需送养 → 工坊态劫持孵化槽/能量截断皆为此打补丁 | 支柱3「零焦虑」（截断把浪费可视化） | A2、A6 |
| 4 | 收集线两周耗尽：3品种一周解锁完、保底不可见、解锁无预告；广告位#1固定3000×3与步数无关 | 支柱1（广告替代走路）、支柱5「制造期待」 | A3 |

**设计目标**（整合方案 §一，C2 为本文档补充认领）：

| # | 目标 | 承接批次 |
|---|------|---------|
| 1 | 孵化循环有决策感，不是自动灌蛋 | **C2**（保底可视化/解锁预告先行，蛋分档待美术） |
| 2 | 低活跃不饿死，高活跃不撑死 | B1 |
| 3 | 能量溢出不焦虑，纯工程收口 | B3 |
| 4 | 工坊是每天的小惊喜，不是溢出出口 | C1 |
| 5 | 包满体验不卡住，玩家知道下一步做什么 | B3 |

**已拍板的设计决策（2026-07-22 制作人）**：
- ✅ 费率翻转/孵化成本曲线/广告步行放大器/经验软拐点/N12/扩容调整——采纳
- ✅ 工坊独立 + 包满引导 + 溢出温和收口（整合方案层2~4）——采纳
- ❌ 送养改"毕业旅行"——**本轮不改**。送养符合现实猫咪救助的「养亲了送好人家」
  语义与现实无奈，"爱心送养"叙事自洽；机制冲突接受并记录。
- ✅ N12 并入本轮；D12/D13 不并入，仅预留接口。
- 📌 棋盘侧 GDD 分叉记账：v3.3 包B 的 K9/K10/K11 与实际落地的 M1~M4 是两条分叉
  迭代线。**以代码为准**，本轮不动棋盘；GDD 下次修订时需正式裁决该分叉。

> 建议顺序：B1 → B2+B3 → B4+C2 → C1 → B5。
> **时序修正（对整合方案的变更）**：原案层3 写"不再需要工坊接包满能量"并把该
> 移除放在 B3，但 C1 标注可并行、排期在后——若 B3 先上线而 C1 未到，线上工坊
> 即成半死系统（旧槽冻结、新盒不产）。修正为：**B3 不触碰工坊路由，旧工坊保持
> 完整功能；能量→工坊路由的移除、旧逻辑清理、存档迁移全部随 C1 同版本上线。**

**设计原则**
1. 步数是唯一核心驱动：任何供给改动后，广告能量占比结构性 ≤50%。
2. 低活跃玩家的核心循环必须可转：孵化节奏 ≥1颗/3天。
3. 工坊/门票/经验/宝箱各自订阅步数信号独立计数，**互不挤占**；池满截断只影响
   能量本身。
4. 高活跃能量回调（9200→6550）属设计意图内的软帽，需版本公告说明+补偿。
5. 所有数值改动双侧同步（GDScript + econ_sim）并全量回归；标 `[待sim校准]` 的
   数值以回归收敛值为准，落地后回填本文档。
6. **B1 的 sim 校准直接采用 C1 的工坊新模型口径**（能量水槽已移除的世界），
   避免 C1 落地后二次调参。

## 2. B1 供给侧热修（P0）

### 2.1 能量费率翻转

**现状**：T1(0–1000)×0.3 / T2(–3000)×1.0 / T3(–5000)×1.2 / T4(5000+)×1.5，
新手期 T1→0.8。边际费率递增对玩家不可见，激励为零；唯一可感知后果是低活跃数字难看。

**新费率**（`[待sim校准]`，目标值优先于系数本身）：

| 段 | 步数区间 | 系数 |
|---|---|---|
| T1 | 0–1500 | **1.1** |
| T2 | 1500–4000 | **1.0** |
| T3 | 4000–6000 | **0.8** |
| T4 | 6000+ | **0.4** |

| 画像 | 旧日能量 | 新日能量 | 蛋节奏（4250） |
|---|---|---|---|
| 低活跃 1500 | 800 | **1650** | 5.3天 → **2.6天** |
| 中活跃 5000 | 4700 | **4950** | ~1天（基本不变） |
| 高活跃 8000 | 9200 | **6550** | 2.2颗/日 → **1.5颗/日** |

新手保护期：原「T1→0.8」废弃（T1 已 1.1），改为**首 7 天全段 ×1.2**。

### 2.2 孵化成本成长曲线

**现状**：所有蛋统一 4250；新手第二颗蛋低活跃要等 3 天+（D2–D4 流失决胜期）。

**新曲线**（`CatData.get_hatch_cost` 增加 hatched_count 参数）：

| 第 N 颗蛋 | 成本 |
|---|---|
| 1 | 教学蛋，直接送满（现状不变） |
| 2 | **1500** |
| 3 | **2500** |
| 4 | **3500** |
| 5–19 | 4250 |
| 20+ | 每 10 颗 +250，封顶 **6000** `[待sim校准]` |

### 2.3 广告位 #1 改步行放大器

单次奖励 = `min(3000, 0.3 × 当日步数能量)`，每日 3 次不变；文案「**今日步行加成**」。
数学保证：3 次总和 ≤ 0.9×步数能量 → 占比恒 ≤47.4%。
- `S06_HatchPage.gd:157` 广告余量回池路径删除（封顶到当前蛋需求）。
- 「去广告¥12」直领政策不变（直领同样按联动公式计算）。
- 钻石能量包（30钻/3000）不动——付费通道保留原价值。

### 2.4 步数里程碑宝箱（为 D12 预留）

当日步数达 3000 / 6000 / 10000 各开一个宝箱（内容 `[待定]`：小额金币 + 互动道具，
不含能量——能量只从费率来）。实现打 `milestone_tag` 字段，未来 D12 三段式日循环
直接挂接。**本批仅做数据结构与结算，演出从简。**

### 2.5 B1 验收

- A3：12 画像全绿（占比 ≤0.5）。
- A6：中活跃截断=0；高活跃日均截断 ≤500 `[待sim校准]`（断言 bug 修复见 B5；
  按 C1 工坊新模型口径回归）。
- 新增 **A12**：低活跃（无广告）孵化节奏 ≥1颗/3天。
- 新增 **A13**：中活跃新手 D2 内可孵出第二颗蛋。
- A7/A8/A11 无新增 FAIL。

## 4. B3 能量溢出收口 + 包满引导（P0，整合方案层2+层3）

### 4.1 能量溢出收口（层2）

```
走路 → 主池(15000) → 池满 → 能量截断，步数照记
                              │
                          HUD「🌸 今日步数能量已满」（当日仅第一次弹出）
                          图标变粉色满溢光泽态（成就态，非警告态）
                          经验+门票+工坊计数 继续正常产生
```

1. 截断时顶部能量图标变**粉色光泽满态**（非红/黄警告色）。
2. Toast 温和、**当日第一次溢满时弹一次**，不反复打扰。
3. 仅能量截断：步数照记、经验照常、门票照产、工坊计数照走（C1 后）。
4. 消耗能量（孵蛋/广告加速）使池 <15000 后恢复产出。

### 4.2 包满引导弹窗（层3）

```
蛋 ready → 玩家点击孵化 → 检测包已满
                            │
                    非阻塞引导弹窗：
                    「花园已住满啦～送养一只猫咪到云领养中心，
                     或扩容猫包，就能迎接新朋友」
                    [去送养] [去扩容] [稍后]
```

- 「去送养」→ 跳转送养流程；「去扩容」→ 跳转商店扩容档位；
- 「稍后」→ 关闭弹窗，蛋槽保持 ready（锁图标），能量积主池、满则截断；
- 送养/扩容后回花园 → 提示「新猫咪在等你孵化！」→ ready 蛋可提取，
  收猫后 `HatchEngine` 自动落新蛋。
- **本批不触碰工坊路由**：包满+无蛋在孵时旧工坊态照旧运转，直至 C1 上线
  （见 §1 时序修正）。

### 4.3 B3 验收

- E-1：池满 HUD 当日仅弹一次、成就态视觉；经验/门票不受影响。
- E-2：包满点击 ready 蛋出引导弹窗、三按钮跳转正确、不卡死不报错；
  送养/扩容后蛋恢复可取、收猫自动落新蛋。
