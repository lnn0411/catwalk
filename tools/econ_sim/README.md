# 经济模拟器 v1.0 — 猫步天下

自动化蒙特卡洛模拟验证 v1.0 全部货币与资源闭环。

## 一键运行

```bash
cd tools/econ_sim

# 运行单个 profile
python run_sim.py --profile medium_ad_off_nocard --days 30 --iterations 100

# 运行全部 12 个 profile × 30/90 天
python run_sim.py --all --iterations 100
```

## 输出目录

- `output/{profile_name}_30d.csv` — 日粒度曲线（P50/P95）
- `output/{profile_name}_90d.csv` — 90天粒度曲线
- `output/assertion_report.md` — A1~A10 断言报告

## 参数配置

所有数值从 `config/params.json` 读取，键名与 GDD v3.1 §18.1 速查表及各章参数表一一对应。

修改参数后重跑即可回归验证。

## GDD 设计意图 vs 模拟实测（P1~P5 大整改后基线，2026-07-23）

历史上本节记录的 A2/A3/A4/A6 四个 FAIL 已在上线整备大整改
（`specs/launch_overhaul_master_plan.md`）中全部转绿，当前 13 画像 × 30/90 天
断言 **A1~A14 零 FAIL**。关键修复与新基线：

### 广告能量占比（A3）✅
广告位#1 改「今日步行加成」= min(3000, 0.3×当日步数能量)——3 次总和
≤0.9×步数能量，占比结构性 ≤47.4%（实测全画像 0.474）。

### 橘猫满级（A4）✅
经验软拐点（≤4000 步全额、超出 50%）后：中活跃 34 天、高活跃 25 天，
均入 30±5。断言仅在 90 天档、中/高活跃评估。满级溢出经验 1000:1 转羁绊点
（N12，日 ≤5）。

### 首档扩容（A2）✅
费率翻转 + 首档 7000 金下，中活跃自然分布 D19~40（月卡提前 ~2 天属付费
权益，断言带宽 [18,42]）；高活跃 [15,40]；低活跃 observe。

### 能量截断（A6）✅
供给修平（中活跃开广告 13700→~9400）+ 工坊改独立步数礼盒（不再依赖能量
水槽）后：中活跃截断=0，高活跃 0 截断天。旧高活跃断言的空转 bug
（0-1 比例误与 days//7 比较恒 PASS）已重写为「截断天占比 ≤1/7 + 日均
截断 ≤500」。

### 新增断言（P1~P5）
- **A12** 低活跃孵化节奏 ≥9 只/30 天（实测 13）
- **A13** 中活跃新手 D2 内孵出第二颗蛋（实测 D1.0）
- **A14** 零送养中活跃月花瓣 ≥150（实测 520~680）
- **A5b** 改跨画像合并统计（有限窗口截尾偏置校准，靶心 0.22±0.08）

### sim 保真度修正记录（以客户端实现为权威）
- 随行猫经验从「轮换最落后」改为固定主力猫（旧模型致 A4 假失败）
- 稀有度保底从「逢 N 必出」改为连续 miss 计数（HatchEngine 同构）
- 品种抽取从加权表改为「已解锁均匀随机 + 5 次未出保底排除」，解锁计数用
  累计孵化数（BreedUnlockEngine 同构；params 的 breed_pool_weights 废弃）
- 送养口径对齐 RelinquishSystem：每次 +50 金、花瓣周限 500 截断
- 能量池封顶移至孵化消耗后（日粒度「先封顶后消耗」制造真实中不存在的截断）
- 工坊为 C1 步数礼盒模型（3000 步/盒、日 ≤3、未开 ≤5、配饰重复折花瓣、
  花卉每种上限 5 超限折 10 花瓣）
