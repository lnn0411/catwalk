# T4 全系列自检（深度版）使用说明

基于 2026-06-22 提交（T4-01 ~ T4-15）生成。文件：
- tests/t4_full_self_check.gd
- tests/t4_full_self_check.tscn

## 与现有 tests/t4_self_check.gd 的区别

现有那份是**浅检**：只查"文件是否存在 / 方法是否存在"（20/20 PASS）。
本份是**深检**，多做两件浅检抓不到的事：

1. **编译错误总闸**：逐个 `load()` 今天新增/改动的 29 个脚本与场景。
   任何一个有编译错误（如 20:26 修过的 BottomNav.TABS NotFound）→ 立即 [XX] 报出。
2. **行为断言**：真正跑引擎逻辑，而非只看 has_method。例如：
   - T4-05：dummy 猫连续 5 次互动 → 断言进入 annoyed，再 reset 验证退出
   - T4-10：roll_gift() 返回值必须命中礼物目录；3槽×3000能量常量校验
   - T4-11：稀有度送养系数必须为 0/1/2/5；周花瓣上限=500
   - T4-13：天气加成返回字典、时段名非空、雨天概率=15%
   - T4-15：determine_breed() 返回值必须是合法品种
   - T4-06：探索槽=2、时长=1/2/4h、slot1 解锁阈值=5

## 运行（headless，退出码 = 失败数，可接 CI）

    godot --headless res://tests/t4_full_self_check.tscn

控制台输出每项 [OK]/[XX]，末尾打印 `X/Y PASS` 与 FAIL 项标签；
退出码为失败数（0 = 全过），方便 CI 判定。

## 覆盖范围

编译总闸 + T4-01~T4-15 全 15 项（含 T4-14b），共 40+ 个断言。

## 注意

- 少数行为用例会写入再 reset（EmotionStateMachine 用 dummy id、WorkshopData.roll 后 reset_pity），
  尽量不污染真实存档；**建议在干净 profile 上跑最干净**。
- 本份与现有浅检可共存，互补。CI 里建议两份都跑。

## 附：棋盘（猫咪合合乐）自检三件套（2026-07-22 起）

棋盘玩法每批交付前必跑，全绿才可提交：

```bash
# 1. 编译扫描（抓类型推断/标识符错误——headless 场景自检覆盖不到的编辑器级错误）
GODOT=/path/to/godot tests/check_scripts.sh

# 2. 规则逻辑自检（棋盘 136 断言 + 等级/里程碑/B6 57 断言）
godot --headless res://tests/board_game_selfcheck.tscn
godot --headless res://tests/level_state_manager_selfcheck.tscn

# 3. 蒙特卡洛整局实测（greedy/casual 双策略分关卡通关率，MC_RUNS 控制局数）
MC_RUNS=1000 godot --headless res://tests/board_montecarlo.tscn
```

改到经济数值（奖励表/门票/里程碑/B6）时追加：

```bash
cd tools/econ_sim && python3 run_sim.py --all --iterations 50
# 检查 output/assertion_report.md 的 A7/A8 无新增 FAIL
```

背景：编译扫描是 PR#2 的教训——`Cannot infer the type of "chosen"` 属于
编辑器级类型错误，gdparse（纯语法）和场景自检（不加载 S14）都抓不到。
check_scripts.sh 会自动过滤 --check-only 不加载 autoload 导致的误报。
